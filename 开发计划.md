# 个人主页开发文档（重制版）

## 1. 目标与范围

- 目标：开发一个配置驱动的个人主页，支持通过配置文件自动生成跳转按钮。
- 视觉：默认采用黑暗风主题，标题使用像素风字体并具备打字机动效。
- 输出：项目构建后产物为纯静态 html/js/css 资源，可直接部署到 Nginx。
- 非目标：不接入后端接口、不依赖数据库、不实现登录鉴权。

## 2. 技术方案

| 维度 | 方案 | 说明 |
| ---- | ---- | ---- |
| 前端框架 | Vue 3 | 组件化清晰，维护成本低 |
| 构建工具 | Vite | 快速开发与静态构建 |
| 语言 | TypeScript | 约束配置模型与组件参数 |
| 动效 | typed.js | 标题打字机效果 |
| 样式 | 原生 CSS + 变量 | 高可控视觉系统，适配参考图风格 |

## 4. 配置驱动设计

页面配置文件路径：`public/config.json`

```json
{
  "pageTitle": "在线服务",
  "titleTyping": ["在线服务", "My Home Services"],
  "subtitle": "配置驱动的个人主页导航",
  "layout": {
    "columns": {
      "desktop": 3,
      "tablet": 2,
      "mobile": 1
    }
  },
  "typewriter": {
    "typeSpeed": 100,
    "backSpeed": 60,
    "backDelay": 1800,
    "startDelay": 250,
    "loop": true,
    "cursorChar": "_"
  },
  "links": [
    {
      "name": "我的笔记",
      "url": "https://example.com/notes",
      "icon": "home",
      "target": "_blank",
      "highlighted": false
    }
  ]
}
```

说明：

- 新增按钮：在 `links` 数组追加对象。
- 删除按钮：移除对应对象。
- 调整布局：修改 `layout.columns`。
- 调整打字机效果：修改 `typewriter` 参数。

## 5. 开发实施步骤

### Step 1: 安装依赖

```bash
pnpm install
```

### Step 2: 本地开发预览

```bash
pnpm dev
```

### Step 3: 调整配置与样式

- 按实际站点信息更新 `public/config.json`。
- 通过 `src/styles/tokens.css` 调整颜色、字体、阴影和圆角变量。
- 若需要替换字体，可在 `tokens.css` 中更改 `--font-title` 与 `--font-body`。

### Step 4: 构建静态资源

```bash
pnpm build
```

构建后输出目录：`dist/`

### Step 5: 本地构建产物验证

```bash
pnpm preview
```

> 注意：请通过 HTTP 服务访问构建产物（如 `pnpm preview` 或 Nginx）。直接用 `file://` 打开 `dist/index.html` 可能出现空白页。

## 6. 验收标准

- [ ] 修改 `public/config.json` 后可自动渲染站点按钮。
- [ ] 标题具备打字机效果，且字体风格符合参考图方向。
- [ ] 页面在桌面与移动端网格布局正常。
- [ ] 执行 `pnpm build` 成功，`dist/` 仅包含静态资源。
- [ ] 项目运行全程不依赖后端接口。

## 7. 部署说明（Nginx）

1. 将 `dist/` 内容发布到 Nginx 静态目录。
2. 配置站点根目录指向 `dist/`。
3. 重载 Nginx 并验证首页、静态资源和链接跳转。

## 8. 风险与回退

- 字体外链不可达：改为本地字体文件并更新 `tokens.css`。
- 配置字段错误：`site-config.ts` 会做兜底处理，避免白屏。
- 动效兼容性问题：系统开启减少动态效果时自动降级为静态标题。
- 部署资源路径异常：回退默认路径策略后重新构建验证。
