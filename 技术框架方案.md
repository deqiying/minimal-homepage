# 个人主页前端技术框架方案

## 1. 项目定位

本项目是一个配置驱动的个人主页导航站，目标是在无后端依赖的前提下，交付可维护、可扩展、可静态部署的前端页面。

## 2. 约束与边界

- 仅前端实现，不包含后端接口、数据库、鉴权系统。
- 数据来源为本地配置文件，导航按钮通过配置生成。
- 构建结果必须是纯静态资源，可直接托管在 Nginx。
- 页面默认采用黑暗风视觉，标题字体风格接近参考图并支持打字机效果。

## 3. 技术选型

| 维度 | 方案 | 说明 |
| ---- | ---- | ---- |
| 框架 | Vue 3 | 组件化清晰，适合中小型项目持续维护 |
| 构建 | Vite | 开发启动快，构建产物为静态 html/js/css |
| 语言 | TypeScript | 降低配置与组件边界错误 |
| 动效 | typed.js | 成熟稳定，打字机参数可控 |
| 样式 | 原生 CSS + 变量体系 | 更贴合本项目视觉定制需求 |

## 4. 核心架构

### 4.1 数据层

以 `public/config.yaml` 作为单一配置源，字段包含：

- `pageTitle`、`titleTyping`、`subtitle`
- `siteIcon.favicon`（浏览器标签页 favicon）
- `layout.columns`（桌面/平板/移动列数）
- `typewriter`（速度、延迟、循环、光标）
- `timeline`（时间、文案、多行内容）
- `links`（按钮名称、URL、图标、目标窗口）

### 4.2 组件层

- `App.vue`：页面编排和配置装配
- `TypewriterTitle.vue`：标题打字机动效和降级处理
- `TimelineSection.vue`：纵向时间线布局与节点卡片
- `InlineRichText.vue`：时间线轻量格式渲染
- `LinkGrid.vue`：响应式网格布局
- `LinkCard.vue`：单卡片展示、悬停态与跳转行为
- `AppIcon.vue`：图标渲染组件

### 4.3 样式层

- `tokens.css`：颜色、字体、阴影、圆角等主题变量
- `base.css`：网格背景、全局排版、动效与可访问性降级
- 高密度重复渲染的卡片层使用静态半透明渐变和阴影表达质感，避免依赖运行时 `backdrop-filter` 造成缩放重绘开销

## 5. 关键设计决策

1. 保持最小依赖，不引入路由和状态管理。
2. 通过 `public/config.yaml` 实现配置驱动，修改配置即可增删时间线和按钮。
3. 时间线内容使用 YAML 块文本，降低多行文案维护成本。
4. 打字机效果遵循系统 `prefers-reduced-motion`，自动降级为静态文本。
5. 采用 CSS 变量统一视觉令牌，降低后续换肤和维护成本。
6. 浏览器 favicon 保持 `index.html` 默认兜底，并在运行时按 `siteIcon.favicon` 动态覆盖，避免首屏图标缺失。
7. 站点图标资源支持扩展备用主题，但不与 `footer.brand.icon` 的页脚品牌图标语义混用。
8. 部署产物采用“稳定页面壳 + 运行时资源清单”模式，降低旧 `index.html` 强缓存继续引用失效 hash 资源的白屏风险。

## 6. 构建与部署策略

- 开发命令：`pnpm dev`
- 构建命令：`pnpm build`
- 预览命令：`pnpm preview`

Vite 构建后输出 `dist/`：

- `index.html`
- `asset-manifest.json`
- `assets/*.js`
- `assets/*.css`
- `config.yaml`（由 public 目录复制）

Nginx 仅需将 `dist/` 目录作为静态站点根目录。

其中：

- `index.html` 作为稳定页面壳，只负责基础 DOM、默认 favicon、字体和 boot loading。
- 页面启动时先请求 `asset-manifest.json`，再动态加载当前版本的入口 JS/CSS。
- `config.yaml` 运行时请求会附带显式 cache-busting 参数，以减轻部署后旧配置长期缓存的问题。

注意：构建产物应通过 HTTP 服务访问。直接使用 `file://` 打开 `dist/index.html` 可能导致资源加载失败而出现空白页。

## 7. 风险与回退

- 外网字体不可达：可切换为本地字体文件。
- 配置格式错误：应用使用默认配置兜底，避免白屏。
- 低性能设备动画卡顿：自动启用 reduced-motion 降级。
- 资源路径异常：回退 Vite 默认路径并重新构建验证。

## 8. 扩展路径

- 增加链接分组和关键字搜索。
- 增加主题切换（浅色/深色）。
- 扩展为多页面时，可平滑引入 Vue Router 或迁移内容层方案。
